실습 목표
1. 서블릿 초기화 파라미터의 사용법
- DB 정보와 같이 배치할 때 변경될 수 있는 정보는 
  소스 코드에 문자열로 입력(하드 코딩; hard coding)해서는 안된다.
  왜냐하면, 나중에 DB 정보가 바뀌면 모든 소스를 변경하고 다시 컴파일해야 하기 때문이다.
- 해결책: 
  외부 파일에 변경할 정보를 둔다.
  web.xml의 서블릿 배치 정보에 DB 관련 정보를 둔다.
- 초기화 파라미터 설정 방법
  애노테이션으로 설정 => 소스 코드에 설정하기 때문에 이전과 다를 바 없다.
  web.xml에 설정 => 추천
